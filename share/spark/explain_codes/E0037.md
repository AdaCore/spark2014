Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   Counter : Integer := 0;

   function Next return Integer
     with Side_Effects, Global => (In_Out => Counter);

   type Int_Fn is access function return Integer;

   Cb : Int_Fn := Next'Access; -- error
end Example;
```

This error is issued on an `'Access` attribute reference whose prefix is a
function annotated with `Side_Effects`. [Section 4.1.4(4) of the SPARK
Reference Manual] requires that the `Side_Effects` aspect of a subprogram used
as the prefix of `'Access` be False.

Functions with side effects modify state beyond their return value, which means
calls through the resulting access value would produce hidden state changes
that GNATprove cannot track at the call site. Because the access value can be
stored and called from any context, the verifier would have no way to know
which global state might be modified by the indirect call. Requiring that the
function have no side effects keeps indirect calls predictable and verifiable.

The fix is to convert the function with side effects into a procedure that
returns its result through an `out` parameter and passes all state through
explicit parameters:

```ada
package Example with SPARK_Mode => On is
   procedure Next (Counter : in out Integer; Result : out Integer);

   type Next_Proc is access procedure
     (Counter : in out Integer; Result : out Integer);

   Cb : Next_Proc := Next'Access;
end Example;
```

```ada
package body Example with SPARK_Mode => On is
   procedure Next (Counter : in out Integer; Result : out Integer) is
   begin
      Counter := Counter + 1;
      Result := Counter;
   end Next;
end Example;
```

[Section 4.1.4(4) of the SPARK Reference Manual]:
