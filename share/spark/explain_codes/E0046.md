Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   type Vol is record
      X : Integer;
   end record with Volatile;

   Obj : Vol with Relaxed_Initialization;  -- error
end Example;
```

This error is issued when a part of an effectively volatile type, or of an
effectively volatile object, is given relaxed initialization. [Section 6.10(7)
of the SPARK Reference Manual] states that no part of an effectively volatile
type, or of an effectively volatile object, shall have relaxed initialization.

The fix is to remove the `Relaxed_Initialization` aspect; the object then follows
the standard data initialization policy:

```ada
package Example with SPARK_Mode => On is
   type Vol is record
      X : Integer;
   end record with Volatile;

   Obj : Vol;
end Example;
```

Relaxed initialization cannot be combined with an effectively volatile type or
object. Any data that must be exempted from the standard initialization policy
has to be held in a type or object that is not effectively volatile.

[Section 6.10(7) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/subprograms.html#relaxed-initialization
