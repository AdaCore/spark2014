Example of illegal code:

```ada
procedure Bad_Intrinsic with SPARK_Mode is

   type Integer_8 is range -128 .. 127;

   type Unsigned_8 is mod 256;

   function "+" (X : Unsigned_8; Y: Integer_8) return Unsigned_8 with
     Import, Convention => Intrinsic;
   --  error: Intrinsic convention on arithmetic operator with unexpected profile is not allowed in SPARK
   --  error: types of operands should match the result type

   function Id (X : Integer_8) return Integer_8 is (X);

   X : Unsigned_8 := 127;
   Y : Integer_8 := Id (-10);
begin
   pragma Assert (X + Y = 117);
end;
```

This error is issued on a function specification that declares an arithmetic
operator with the ``Intrinsic`` convention when its profile does not match the
profile of a predefined operator. For example, in the code above, ``"+"`` does
not match any predefined operator as the type of its right operand does not
match its result type. This violation is only issued on functions that are
called from SPARK code. These declarations are forbidden in SPARK, as their
meaning is compiler-dependent and might introduce conversions under the hood.
