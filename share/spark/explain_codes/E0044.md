Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   type T is tagged record  -- error
      F : Integer;
   end record with Relaxed_Initialization;
end Example;
```

This error is issued when a part of a tagged type, or of a tagged object, is
given relaxed initialization. [Section 6.10(6) of the SPARK Reference Manual]
states that no part of a tagged type, or of a tagged object, shall have relaxed
initialization.

The fix is to remove the `Relaxed_Initialization` aspect; the tagged type then
follows the standard data initialization policy:

```ada
package Example with SPARK_Mode => On is
   type T is tagged record
      F : Integer;
   end record;
end Example;
```

Relaxed initialization cannot be combined with a tagged type. Any data that must
be exempted from the standard initialization policy has to be held in a type
that is not part of a tagged type.

[Section 6.10(6) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/subprograms.html#relaxed-initialization
