Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   type I is new Integer with Relaxed_Initialization;

   type R is tagged record
      F : I;  -- OK: component of the root type
   end record;

   type C is new R with record
      G : I;  -- error: component of a tagged extension
   end record;

   X : R with Relaxed_Initialization;  -- error: tagged object

   type D is new R with null record
     with Relaxed_Initialization;  -- error: tagged type
end Example;
```

This error is issued when a tagged type or a tagged object is given relaxed
initialization, as with `X` and `D` here, or when a component declared in a
tagged extension has relaxed initialization, as with `G` here. [Section 6.10(6)
of the SPARK Reference Manual] states that no part of a tagged type, or of a
tagged object, shall have relaxed initialization. A component with relaxed
initialization declared in the root type of a tagged hierarchy is nevertheless
accepted, as with `F` here: GNATprove needs to know that a component exists in
order to handle its initialization, which is the case for components of the
root type but not for components of a (potentially hidden) extension.

For a component of a tagged extension, the fix is to declare the component in
the root type of the hierarchy instead:

```ada
package Example with SPARK_Mode => On is
   type I is new Integer with Relaxed_Initialization;

   type R is tagged record
      F : I;
      G : I;
   end record;

   type C is new R with null record;
end Example;
```

For a tagged type or a tagged object, the fix is to remove the
`Relaxed_Initialization` aspect; the type or object then follows the standard
data initialization policy:

```ada
package Example with SPARK_Mode => On is
   type I is new Integer with Relaxed_Initialization;

   type R is tagged record
      F : I;
   end record;

   type D is new R with null record;

   X : R;
end Example;
```

Relaxed initialization cannot be applied to a tagged type or a tagged object as
a whole, nor to a component declared in a tagged extension. Any data that must
be exempted from the standard data initialization policy has to be held in a
component of the root type of the hierarchy, or outside the tagged hierarchy
altogether.

[Section 6.10(6) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/subprograms.html#relaxed-initialization
