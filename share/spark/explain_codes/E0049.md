Example of illegal code:

```ada
pragma Extensions_Allowed (All_Extensions);

procedure Example with SPARK_Mode is
   X : Integer := 0;
begin
   for I in 1 .. 10 loop
      X := X + 1;
      <<L>>
      X := X + 1;
      pragma Loop_Invariant (X = I + 1);
      X := X'At (L);
   end loop;
end Example;
```

This error is issued on a reference to the At attribute if there is a
Loop_Invariant or Loop_Variant pragma between the label and the At attribute
reference. [Section 4.1.4.1(12) of the SPARK Reference Manual] forbids this
situation unless the pragma is in a nested loop.
Indeed, the Loop_Variant or Loop_Invariant pragma introduces a cutpoint that
causes GNATprove to forget all information about the statement list including
the referenced location.

The fix is to introduce a variable that can be used to expand the reference
attribute through an assignment at the location of the label:

```ada
pragma Extensions_Allowed (All_Extensions);

procedure Example with SPARK_Mode is
   X   : Integer := 0;
   X_L : Integer;
begin
   for I in 1 .. 10 loop
      X := X + 1;
      <<L>>
      X_L := X;
      X := X + 1;
      pragma Loop_Invariant (X = I + 1 and X_L = X - 1);
      X := X_L;
   end loop;
end Example;
```

[Section 4.1.4.1(12) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/names-and-expressions.html#attribute-at
