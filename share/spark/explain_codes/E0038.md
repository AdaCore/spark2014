Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   Sensor : Integer with Volatile, Async_Writers;

   function Read_Sensor return Integer
     with Volatile_Function, Global => (Input => Sensor);

   type Sensor_Fn is access function return Integer;

   Reader : Sensor_Fn := Read_Sensor'Access; -- error
end Example;
```

or, if the function is volatile because it includes a volatile formal
parameter:

```ada
package Example_Alt with SPARK_Mode => On is
   type Sensor is record
      Value : Integer;
   end record with Volatile;

   function Read (S : Sensor) return Integer
     with Volatile_Function;

   type Sensor_Fn is access function (S : Sensor) return Integer;

   Reader : Sensor_Fn := Read'Access; -- error
end Example_Alt;
```

This error is issued on an `'Access` attribute reference whose prefix is a
volatile function. [Section 4.1.4(3) of the SPARK Reference Manual] requires
that the `Volatile_Function` aspect of a function used as the prefix of
`'Access` be False.

The fix is to separate the volatile read from the logic that processes the
result. Read the volatile object directly at the call site and pass the value
to a non-volatile function through a parameter:

```ada
package Example with SPARK_Mode => On is
   Sensor : Integer with Volatile, Async_Writers;

   function Process (Raw : Integer) return Integer;

   type Process_Fn is access function (Raw : Integer) return Integer;

   Processor : Process_Fn := Process'Access;
end Example;
```

```ada
package body Example with SPARK_Mode => On is
   function Process (Raw : Integer) return Integer is
   begin
      return Raw + 1;
   end Process;
end Example;
```

The caller reads `Sensor` directly and passes its value to `Processor.all`,
keeping the volatile access visible to GNATprove.

If the function is volatile because it includes a volatile formal parameter, an
alternative workaround is to convert the function into a procedure:

```ada
package Example_Alt with SPARK_Mode => On is
   type Sensor is record
      Value : Integer;
   end record with Volatile;

   procedure Read (S : Sensor; Result : out Integer);

   type Sensor_Proc is access procedure (S : Sensor; Result : out Integer);

   Reader : Sensor_Proc := Read'Access;
end Example_Alt;
```

```ada
package body Example_Alt with SPARK_Mode => On is
   procedure Read (S : Sensor; Result : out Integer) is
   begin
      Result := S.Value;
   end Read;
end Example_Alt;
```

The procedure passes the result through an `out` parameter instead of a return
value. Because procedures do not carry the `Volatile_Function` aspect, the
restriction does not apply.

[Section 4.1.4(3) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/names-and-expressions.html#attributes
