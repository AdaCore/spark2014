Example of illegal code:

```ada
pragma Profile (Ravenscar);
pragma Partition_Elaboration_Policy (Sequential);

package Example with SPARK_Mode => On is
   protected Obj is
      function Get return Integer;
   private
      Value : Integer := 0;
   end Obj;
end Example;
```

```ada
package body Example with SPARK_Mode => On is
   protected body Obj is
      function Get return Integer is
         type Int_Fn is access function (X : Integer) return Integer;

         function Helper (X : Integer) return Integer is (X + Value);

         Cb : Int_Fn := Helper'Access; -- error
      begin
         return Value;
      end Get;
   end Obj;
end Example;
```

This error is issued on an `'Access` attribute reference whose prefix is a
subprogram declared within a protected type or protected object. [Section
4.1.4(2) of the SPARK Reference Manual] forbids taking `'Access` of such
subprograms.

The fix is to move the subprogram outside the protected object so that any
data it needs is passed explicitly through parameters:

```ada
pragma Profile (Ravenscar);
pragma Partition_Elaboration_Policy (Sequential);

package Example with SPARK_Mode => On is
   type Int_Fn is access function (X : Integer) return Integer;

   function Helper (X : Integer) return Integer;

   protected Obj is
      function Get return Integer;
   private
      Value : Integer := 0;
   end Obj;
end Example;
```

```ada
package body Example with SPARK_Mode => On is
   function Helper (X : Integer) return Integer is (X + 1);

   protected body Obj is
      function Get return Integer is
         Cb : Int_Fn := Helper'Access;
      begin
         return Cb.all (Value);
      end Get;
   end Obj;
end Example;
```

[Section 4.1.4(2) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/names-and-expressions.html#attributes
