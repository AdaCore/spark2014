Example of illegal code:

```ada
with Ada.Finalization;

type My_Resource is new Ada.Finalization.Controlled with record  -- error
   Handle : Integer;
end record;
```

This error is issued on declarations of controlled types, i.e., types that
derive from `Ada.Finalization.Controlled` or
`Ada.Finalization.Limited_Controlled`, and on declarations of composite types
that have a component of such a type. As stated in [section 7.6 of SPARK
Reference Manual], controlled types are not allowed in SPARK because they rely
on compiler-inserted implicit calls to `Initialize`, `Adjust`, and `Finalize`
subprograms. These hidden calls cannot be reasoned about by GNATprove, as SPARK
requires all side effects to be explicit in contracts. Additionally, the timing
and ordering of finalization depend on choices made by the compiler, making
controlled types a source of non-portable behavior.

Rewriting controlled types in SPARK is generally difficult. The most practical
workaround is to encapsulate the controlled type in a private section of a
package that is excluded from SPARK analysis using `SPARK_Mode => Off`:

```ada
with Ada.Finalization;

package Resource_Manager
  with SPARK_Mode => On
is
   type My_Resource is limited private;

   procedure Create (R : out My_Resource);
   procedure Destroy (R : in out My_Resource);

private
   pragma SPARK_Mode (Off);

   type My_Resource is new Ada.Finalization.Limited_Controlled with record
      Handle : Integer;
   end record;

   overriding procedure Finalize (R : in out My_Resource);
end Resource_Manager;
```

With this pattern, the controlled type's implementation is hidden from
GNATprove. Client code that interacts with `My_Resource` only through the
public interface can still be fully analyzed in SPARK.

Client code that interacts with My_Resource only through the public interface
can still be fully analyzed in SPARK. However, the programmer must make sure
that the implicit calls do not interfere with the SPARK analysis.

[section 7.6 of SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/packages.html#assignment-and-finalization
