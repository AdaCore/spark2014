Example of illegal code:

```ada
with System;
package P is
   C : constant Integer := 42;
   X : Integer with Address => C'Address; -- error: X is mutable, C is constant
end P;
```

This error is issued on the declaration of an object whose address is specified
to be the address of part of another object (an overlay), when one of the two
objects is constant and the other is mutable. SPARK RM 13.7(4) requires that
two overlaid objects are either both constant or both mutable.`gnatprove` does
not permit modification of constants.

If the intent is to create a read-only view of a mutable object, use an
`aliased` variable together with an `access constant` pointer. Note that
SPARK RM 3.10(5) forbids anonymous access objects at library level, so this
pattern must appear inside a subprogram, entry, or block:

```ada
procedure Example is
   X : aliased Integer := 42;
   C : access constant Integer := X'Access; -- read-only view of X
begin
   null;
end Example;
```

Alternatively, if the goal is to reinterpret the bits of an existing object as
a different type, pass the object as an `in` parameter and declare a `constant`
overlay inside the subprogram. An `in` parameter is constant in SPARK's view,
so a `constant` overlay satisfies the same-constancy requirement of SPARK RM
13.7(4):

```ada
procedure Example (X : in Integer) is
   type Alias_Int is new Integer;
   C : constant Alias_Int with Address => X'Address, Import;
begin
   null;
end Example;
```

For more information, see [section 13.7 of SPARK Reference Manual].
