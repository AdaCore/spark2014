Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   type Arr is array (1 .. 10) of Integer with Relaxed_Initialization;

   type D;
   type D_Acc is access D;

   type Vol is record
      Link : D_Acc;
   end record with Volatile;

   type D is record  -- error
      F : Arr;
   end record;
end Example;
```

This error is issued when a part of a type or object that is tagged, an
`Unchecked_Union`, or effectively volatile is given relaxed initialization.
|SPARK| forbids relaxed initialization on any part of a tagged type, of an
`Unchecked_Union` type, and of an effectively volatile type or object. GNATprove
reports this combined wording, rather than naming the single category, when it
detects the violation in a context where the specific category cannot be
isolated — for instance, in the definition above, where the array type `Arr`
(which has relaxed initialization) is the component `F` of `D`, and `D` is in
turn reachable through the access type `D_Acc` from the effectively volatile
type `Vol`.

The fix is to remove the `Relaxed_Initialization` aspect from the part that lies
within the tagged, `Unchecked_Union`, or effectively volatile type. The type
then follows the standard data initialization policy:

```ada
package Example with SPARK_Mode => On is
   type Arr is array (1 .. 10) of Integer;

   type D;
   type D_Acc is access D;

   type Vol is record
      Link : D_Acc;
   end record with Volatile;

   type D is record
      F : Arr;
   end record;
end Example;
```
