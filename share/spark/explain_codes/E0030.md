Example of illegal code:

```ada
package P with SPARK_Mode => On is

   Offset : Natural := 0;

   type Container is new Natural
     with Iterable => (First       => First,  -- error: First reads global Offset
                       Next        => Next,
                       Has_Element => Has_Element);

   function First       (X : Container) return Natural
     is (Offset);
   function Next        (X : Container; Y : Natural) return Natural
     is (Y + 1);
   function Has_Element (X : Container; Y : Natural) return Boolean
     is (Y < Natural (X));

end P;
```

This error is issued on a function named in the `Iterable` aspect of a
container type (i.e., one of `First`, `Next`, `Last`, `Previous`,
`Has_Element`, `Element`, or `Constant_Reference`) when that function reads
or writes a global variable, either through an explicit `Global` (or
`Depends`) contract or through globals inferred by flow analysis.

[Section 5.5.2(12) of the SPARK Reference Manual] forbids this because the
generalized iteration scheme defined by the `Iterable` aspect calls these
functions implicitly from a `for ... of` loop, and `gnatprove` reasons about
the iteration in terms of the container value alone, currently.

The fix is to make the global data part of the container itself. For example:

```ada
package P with SPARK_Mode => On is

   type Container is record
      Offset : Natural;
      Size   : Natural;
   end record
     with Iterable => (First       => First,
                       Next        => Next,
                       Has_Element => Has_Element);

   function First       (X : Container) return Natural
     is (X.Offset);
   function Next        (X : Container; Y : Natural) return Natural
     is (Y + 1);
   function Has_Element (X : Container; Y : Natural) return Boolean
     is (Y < X.Offset + X.Size);

end P;
```

[Section 5.5.2(12) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/statements.html#generalized-loop-iteration
