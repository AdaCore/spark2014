Example of illegal code:

```ada
pragma Extensions_Allowed (All_Extensions);

procedure Example with SPARK_Mode is
   X : Integer;
begin
   X := 0;
   <<L>>
   X := 1;
   pragma Assert_And_Cut (X = 1);
   X := X'At (L);
end Example;
```

This error is issued on a reference to the At attribute if there is an
Assert_And_Cut pragma in the innermost sequence of statements enclosing the
referenced label between the label and the At attribute reference. This
situation is forbidden by [Section 4.1.4.1(11) of the SPARK Reference Manual].
Indeed, the Assert_And_Cut pragma introduces a cutpoint that causes
GNATprove to forget all information about the statement list including the
referenced location.

The fix is to introduce a variable that can be used to expand the reference
attribute through an assignment at the location of the label:

```ada
pragma Extensions_Allowed (All_Extensions);

procedure Example with SPARK_Mode is
   X   : Integer;
   X_L : Integer;
begin
   X := 0;
   <<L>>
   X_L := X;
   X := 1;
   pragma Assert_And_Cut (X = 1 and X_L = 0);
   X := X_L;
end Example;
```

[Section 4.1.4.1(11) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/names-and-expressions.html#attribute-at
