Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   Counter : Integer := 0;

   procedure Increment with Global => (In_Out => Counter);

   type Callback is access procedure;

   Cb : Callback := Increment'Access; -- error
end Example;
```

This error is issued on an `'Access` attribute reference whose prefix is a
subprogram that has global inputs or outputs. [Section 4.1.4(7) of the SPARK
Reference Manual] requires that a subprogram used as the prefix of `'Access`
have no global effects.

The restriction exists because GNATprove must be able to verify the correctness
of calls made through access-to-subprogram values. When a subprogram has global
effects, the verifier would need to track which globals might be read or
written at every indirect call site. Because the actual target of the call is
only known at run time, this tracking cannot be done precisely, which would
compromise the soundness of the analysis. Requiring the subprogram to be free
of global effects ensures that indirect calls do not introduce hidden data
dependencies or aliasing.

The fix is to replace global variable accesses with explicit parameters, so
that all data flow is visible in the subprogram's signature:

```ada
package Example with SPARK_Mode => On is
   type Callback is access procedure (X : in out Integer);

   procedure Increment (X : in out Integer);

   Cb : Callback := Increment'Access;
end Example;
```

[Section 4.1.4(7) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/names-and-expressions.html#attributes
