Example of illegal code:

```ada
package Example with SPARK_Mode => On is

   V : Integer;
   G : Integer;

   procedure Add_V_To_G with
     Global => (Input => V, In_Out => G),
     Modifies => (V, G);

end Example;
```

This error is issued on a clause of a Modifies aspect which mentions an object
that is not an output of the subprogram. [Section 6.1.14(1) of the SPARK
Reference Manual] requires that all objects mentioned in clauses of the
Modifies aspect of the subprogram are outputs of this subprogram. It can occur
in particular on subprograms without a body in SPARK if they don't have an
explicit Global contract and the inferred one does not contain the expected
output:

```ada
package Example with SPARK_Mode => On is

   V : Integer;
   G : Integer;

   procedure Add_V_To_G with
     Import,
     Modifies => G;

end Example;

In this case, the fix is to add an explicit Globals contract to the subprogram:

```ada
package Example with SPARK_Mode => On is

   V : Integer;
   G : Integer;

   procedure Add_V_To_G with
     Import,
     Global => (Input => V, In_Out => G),
     Modifies => G;

end Example;
```

[Section 6.1.14(1) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/subprograms.html#modifies-aspects
