Example of illegal code:

```ada
procedure Bad with SPARK_Mode => On is
   type Arr is array (1 .. 10) of Integer;
   X : Arr;
begin
   X := (others => 0);
   pragma Assert (X'Initialized);  -- error
end Bad;
```

This error is issued on a reference to the `'Initialized` attribute whose prefix
denotes an object that does not have relaxed initialization. [Section 6.10(4) of
the SPARK Reference Manual] defines the `X'Initialized` attribute only for a
prefix `X` that denotes an object which has relaxed initialization. Here `X` is
an ordinary array object subject to the standard data initialization policy,
so the attribute does not apply to it.

The fix is to give the prefix relaxed initialization, by annotating the object
or its type with the `Relaxed_Initialization` aspect. The `'Initialized`
attribute is then available to query whether the object is fully initialized:

```ada
procedure Good with SPARK_Mode => On is
   type Arr is array (1 .. 10) of Integer;
   X : Arr with Relaxed_Initialization;
begin
   X := (others => 0);
   pragma Assert (X'Initialized);
end Good;
```

Alternatively, if the object does not need relaxed initialization, the fix is
to remove the reference to the `'Initialized` attribute. Under the standard
[Data Initialization Policy of the SPARK User's Guide], GNATprove verifies
that the object is completely initialized before it is read, so there is no
need to query or assert this property:

```ada
procedure Good with SPARK_Mode => On is
   type Arr is array (1 .. 10) of Integer;
   X : Arr;
begin
   X := (others => 0);
end Good;
```
