Example of illegal code:

```ada
procedure P is
   X : Integer := 0;
begin
   <<Start>>             -- error: target of a backward goto
   X := X + 1;
   if X < 10 then
      goto Start;        -- error: backward goto statement
   end if;
end P;
```

This error is issued on a `goto` statement whose target label appears earlier
in the sequence of statements. SPARK RM 5.8(1) requires that a `goto_statement`
appear before its target label in the innermost enclosing sequence of
statements, so only forward gotos are permitted.

The restriction exists because a backward goto creates an implicit loop
structure. SPARK's formal verification techniques rely on loops being
explicitly identified as `loop` statements, so that loop invariants and
loop variants can be attached to them. A backward goto creates a jump back
to an earlier point in the code without such a syntactic marker, making
it impossible for `gnatprove` to apply these techniques automatically and
reason about termination or invariants.

The fix is to replace the backward goto with an explicit loop construct,
which can also be given invariants and variants if needed:

```ada
procedure P is
   X : Integer := 0;
begin
   loop
      X := X + 1;
      exit when X >= 10;
   end loop;
end P;
```

For more information, see [section 5.8 of the SPARK Reference Manual].
