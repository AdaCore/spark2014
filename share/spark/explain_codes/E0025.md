Example of illegal code:

```ada
procedure P is
begin
   ...
exception
   when E : Constraint_Error => -- error
      null;
end P;
```

This error is issued on exception handlers that use a choice parameter (the
`E :` syntax before the exception name). SPARK does not allow choice parameters
in exception handlers because the `Ada.Exceptions.Exception_Occurrence` type
used to represent exception occurrences is not in SPARK, so it cannot be used
in verification. The restriction ensures that the handler body can be analyzed
by GNATprove without requiring support for this type.

The fix is to remove the choice parameter from the handler:

```ada
procedure P is
begin
   ...
exception
   when Constraint_Error =>
      null;
end P;
```

See [section 11.2 of SPARK Reference Manual].

[section 11.2 of SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/exceptions.html#exception-handlers
