Example of illegal code:

```ada
package Example with SPARK_Mode => On is
   type R is record
      I : Integer;
   end record;

   type Root is tagged null record;

   procedure P (X : Root; Y : out R)  -- error
     with Import, Relaxed_Initialization => Y;
end Example;
```

This error is issued on a `Relaxed_Initialization` aspect that applies to a
formal parameter or the result of a dispatching operation, that is, a primitive
subprogram of a tagged type. [Section 6.10(10) of the SPARK Reference Manual]
states that a formal parameter of a dispatching operation shall not have relaxed
initialization, and that the result of a dispatching function shall not have
relaxed initialization.

Here `P` is a primitive of the tagged type `Root`, so it is a dispatching
operation; its parameter `Y` therefore cannot carry the `Relaxed_Initialization`
aspect.

The fix is to specify relaxed initialization on the type of the parameter
instead of on the subprogram: introduce a derived type with the
`Relaxed_Initialization` aspect and use it as the type of `Y`. Objects of the
derived type are handled with relaxed initialization, and the aspect no longer
appears on the dispatching operation, which remains a primitive of `Root`:

```ada
package Example with SPARK_Mode => On is
   type R is record
      I : Integer;
   end record;

   type R_Relaxed is new R with Relaxed_Initialization;

   type Root is tagged null record;

   procedure P (X : Root; Y : out R_Relaxed)
     with Import;
end Example;
```

Alternatively, if dispatching is not needed, the fix is to make the subprogram
non-dispatching. A subprogram that is not a primitive of a tagged type is not a
dispatching operation, so its parameters and result may use relaxed
initialization. Removing the controlling parameter (or declaring the subprogram
where it is not a primitive) lifts the restriction:

```ada
package Example with SPARK_Mode => On is
   type R is record
      I : Integer;
   end record;

   procedure P (Y : out R)
     with Import, Relaxed_Initialization => Y;
end Example;
```

[Section 6.10(10) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/subprograms.html#relaxed-initialization
