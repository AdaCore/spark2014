Example of illegal code:

```ada
package Example with SPARK_Mode => On is

   type R is record
      F, G : Integer;
   end record;

   X : R with Volatile, Async_Writers => True;

   procedure Swap_X_And_Y (Y : out R) with
     Global => (In_Out => X),
     Modifies => Y;

   procedure Swap_X_F_And_Y (Y : out Integer) with
     Global => (In_Out => X),
     Modifies => (Y, X.F);

end Example;
```

This error is issued on the Modifies aspect of a subprogram that has an output
that is effectively volatile for reading and does not mention it, or does
not mention parts of it in its Modifies aspect. [Section 6.1.14(2) of the SPARK
Reference Manual] requires that all the outputs of a subprogram that are
effectively volatile for reading are mentioned alone in a clause without a
guard. Indeed, as objects that are
effectively volatile for reading are considered to potentially be modified
asynchronously during the subprogram, it is not possible to ensure that
they are (partially) preserved.

The fix is to add them to the Modifies aspect:

```ada
package Example with SPARK_Mode => On is

   type R is record
      F, G : Integer;
   end record;

   X : R with Volatile, Async_Writers => True;

   procedure Swap_X_And_Y (Y : out R) with
     Global => (In_Out => X),
     Modifies => (Y, X);

   procedure Swap_X_F_And_Y (Y : out Integer) with
     Global => (In_Out => X),
     Modifies => (Y, X);

end Example;

[Section 6.1.14(2) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/subprograms.html#modifies-aspects
