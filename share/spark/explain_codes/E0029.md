Example of illegal code:

```ada
package P with SPARK_Mode => On is

   type Container is private;

   function First       (X : Container) return Boolean;
   function Next        (X : Container; Y : Boolean) return Boolean;
   function Has_Element (X : Container; Y : Boolean) return Boolean;

private

   type Container is new Boolean
     with Iterable => (First       => First,  -- error: Iterable aspect on full
                       Next        => Next,   --        view of private type
                       Has_Element => Has_Element);
end P;
```

This error is issued on the full view of a private type that carries an
`Iterable` aspect when its partial view does not. The `Iterable` aspect
defines the generalized iteration scheme (`for ... of` loops) for the type,
and clients of the package must be able to see that the type is iterable
in order to use it.

[Section 5.5.2(13) of the SPARK Reference Manual] forbids declaring the
`Iterable` aspect on the full view of a private type. `gnatprove` does not
support this placement, currently.

The fix is to move the `Iterable` aspect to the partial view of the
private type. The functions named in the aspect can still be declared in
the visible part of the package and given their bodies in the package body
or in terms of the full view:

```ada
package P with SPARK_Mode => On is

   type Container is private
     with Iterable => (First       => First,
                       Next        => Next,
                       Has_Element => Has_Element);

   function First       (X : Container) return Boolean;
   function Next        (X : Container; Y : Boolean) return Boolean;
   function Has_Element (X : Container; Y : Boolean) return Boolean;

private

   type Container is new Boolean;

end P;
```

[Section 5.5.2(13) of the SPARK Reference Manual]:
https://docs.adacore.com/live/wave/spark2014/html/spark2014_rm/statements.html#generalized-loop-iteration
