Example of illegal code:

```ada
pragma Extensions_Allowed (All_Extensions);

procedure Example with SPARK_Mode is
   type Int_Acc is access Integer;
   G : Integer := 12;
   function Create return Int_Acc is (new Integer'(G));

   X : Int_Acc;
begin
   <<Init>>
   G := 0;
   X := Create'At (Init);
end Example;
```

This error is issued on a reference to the At attribute whose prefix is
subject to ownership - and a call to a non-traversal function, as per
[Section 3.1O(13) of the SPARK Reference Manual].
[Section 4.1.4.1(10) of the SPARK Reference Manual] states that references to
the At attribute whose prefix is subject to ownership shall only occur inside
an assertion or a constant part of an object. This rule is necessary so
that the implicit object declared by such a reference is never modified. It is
important as this same object could be used multiple times, for example if the
reference occurs in a loop, creating aliasing:

```ada
pragma Extensions_Allowed (All_Extensions);

procedure Bad_Example with SPARK_Mode is
   type Int_Acc is access Integer;
   G : Integer := 12;
   function Create return Int_Acc is (new Integer'(G));
   type Int_Acc_Arr is array (1 .. 10) of Int_Acc;

   A : Int_Acc_Arr;
begin
   <<Init>>
   G := 0;
   for I in 1 .. 10 loop
      A (I) := Create'At (Init);
   end loop;
   --  All cells of A designate the same data
end Bad_Example;
```

The fix is to expand the reference attribute by introducing an object
declaration at the location of the label. It will not apply if the object is
used several times like in Bad_Example:

```ada
pragma Extensions_Allowed (All_Extensions);

procedure Example with SPARK_Mode is
   type Int_Acc is access Integer;
   G : Integer := 12;
   function Create return Int_Acc is (new Integer'(G));
   type Int_Acc_Arr is array (1 .. 10) of Int_Acc;

   X : Int_Acc;
begin
   <<Init>>
   declare
      Create_Init : constant Int_Acc := Create;
   begin
      G := 0;
      X := Create_Init;
   end;
end Example;
```
